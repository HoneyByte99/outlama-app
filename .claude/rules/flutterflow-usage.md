# FlutterFlow Usage

The outalma_service_app project is a speed asset.

Use it as a source for:
- screen inventory,
- user flows,
- design tokens,
- reusable UI patterns,
- loading / empty / error states,
- navigation ideas,
- Firestore schema comparison.

Do not blindly port generated code.
Preferred strategy:
1. inspect the FlutterFlow screen,
2. extract the UX intent,
3. rebuild cleanly in the target architecture,
4. map it to canonical domain models and repository contracts.
