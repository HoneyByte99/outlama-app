# Git Workflow

## Principles
- Small commits
- One logical change per commit
- Clear commit messages
- No mixed concerns without reason
- Push directly after committing — no confirmation needed, user is solo on the project

## Commit process
1. Explain planned change
2. Implement narrowly
3. Run validation
4. Summarize impact
5. Commit and push — no "should I push?" question

## Commit examples
- `feat(provider): add provider profile repository implementation`
- `fix(booking): align enum serialization with functions contract`
- `test(chat): add access control coverage after acceptance`
- `docs(mvp): define scope and delivery standards`
