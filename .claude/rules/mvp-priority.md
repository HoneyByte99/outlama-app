# MVP Priority Rule

Outalma is in MVP mode.

When making tradeoffs:
- prefer shipping the smallest robust version,
- avoid features that do not improve the main booking loop,
- defer sophistication unless it clearly reduces risk or saves major rework.

Always classify work as one of:
- MVP-critical
- quality-critical
- useful but deferrable
- not for MVP

If a task is not MVP-critical, avoid letting it block delivery.
