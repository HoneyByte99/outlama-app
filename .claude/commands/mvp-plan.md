When the user runs `/mvp-plan`, do the following:

1. Inspect the relevant code and docs.
2. State what already exists.
3. State what is missing.
4. Produce a strict MVP-first execution plan.
5. Split the plan into:
   - do now
   - do next
   - defer
   - not for MVP
6. Include validation and test expectations.
7. End with the best first slice to implement.
